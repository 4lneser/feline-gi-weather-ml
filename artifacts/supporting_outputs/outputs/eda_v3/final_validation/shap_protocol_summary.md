# SHAP Protocol Summary

Read-only method verification from `/Users/omaralneser/Documents/TYP/run_eda_v3.py`.

## 1) SHAP data slice
- SHAP is computed on the **validation fold only** (`X_test`) inside binary CV loop.
- Pipeline flow in each fold: fit on `X_train`, then transform `X_test`, then run `TreeExplainer` on transformed validation matrix.
- It is **not** computed on train+validation combined.

## 2) SHAP aggregation statistic
- Per fold and per feature: `mean_abs_shap_fold = mean(abs(shap_values))` over validation samples in that fold.
- Across folds: grouped by `disease × feature_block × feature`, then aggregated as:
  - `mean_abs_shap` = mean of fold-level mean absolute SHAP values
  - `std_abs_shap` = std across folds
- Top-20 tables are ranked by descending `mean_abs_shap` within each `disease × feature_block`.

## 3) Stability metrics
- Fold rankings are generated by ranking `mean_abs_shap_fold` within each `disease × feature_block × fold`.
- Stability metrics computed across fold pairs:
  - Pairwise **Spearman rank correlation** on union of top-50 features (missing rank imputed as 51).
  - Pairwise **Jaccard overlap @10**.
  - Pairwise **Jaccard overlap @20**.
- Reported outputs are fold-pair means/stds per `disease × feature_block`.

## 4) Environmental-only frame consistency
- Environmental-only SHAP runs use the same globally filtered environmental numeric frame (`kept_weather_numeric`) with **31** kept weather numeric features in this run.
- This frame is injected into `feature_blocks["environmental_only"]["numeric"]` before binary CV/SHAP execution, so SHAP uses the same feature block definition as the binary models.
